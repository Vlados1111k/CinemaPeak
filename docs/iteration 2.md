# Звіт за ітерацію №2 (Лабораторна 35)

## Виконані завдання:
1. **Business Logic & LINQ**: Реалізовано перевірку на конфлікт місць (заборона подвійного бронювання) за допомогою LINQ-запиту `.Any()`.
2. **Persistence (JSON)**: Впроваджено асинхронне збереження даних у файл `tickets.json` через узагальнений клас `JsonDataStore<T>`.
3. **Pattern Strategy**: Реалізовано гнучку систему знижок (Student, NoDiscount), що дозволяє змінювати логіку ціноутворення без модифікації сервісів.

## Виконані Use Cases:
- Реєстрація бронювання з автоматичною валідацією.
- Перегляд аналітики (загальна каса, статистика по типах квитків).
- Автоматичне відновлення стану системи при запуску.

## Бізнес-правила:
- **Унікальність місця**: Координати (Ряд/Місце) мають бути унікальними в межах репозиторію.
- **Валідація входу**: Ціна, номер ряду та місця не можуть бути менше або дорівнювати 0 (викидається `ArgumentException`).
- **Ціноутворення**: VIP-квитки мають націнку 50% від базової вартості.
