# Vision: Система бронювання CinemaPeak

## Проблема
Користувачам важко відстежувати наявність вільних місць у кінотеатрі та купувати квитки без черг.

## Цільові користувачі
- Глядачі (хочуть купити квиток).
- Касири/Адміни (хочуть бачити статистику).

## Сценарії використання (Use Cases)
1. **Бронювання квитка:** Користувач обирає місце, система розраховує ціну (Standard/VIP) і зберігає замовлення.
2. **Перегляд розкладу:** Відображення списку доступних фільмів та їх тривалості.
3. **Скасування замовлення:** Можливість видалити квиток з бази (реалізація в наступних ітераціях).

## Нефункціональні вимоги
- **Код:** Використання SOLID та Clean Architecture.
- **Надійність:** Валідація даних у конструкторах (не можна створити фільм на 0 хвилин).
- **Тестованість:** Покриття бізнес-логіки юніт-тестами.
